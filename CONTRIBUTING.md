# Contributing to the Claude Code Codex Plugin

Open an issue for Claude Code integration bugs, Codex plugin improvement ideas, documentation gaps,
installer problems, MCP server behavior, or Claude CLI compatibility reports. For code
changes, describe the problem first and keep pull requests narrowly focused.

Before opening a pull request:

1. Run `npm run verify` from the repository root.
2. Run the plugin validator.
3. Do not add credentials, authentication tokens, or machine-specific paths to committed files.
4. Explain any security impact in the pull request description.

Changes that expand process execution, Claude Code permission modes, MCP tool behavior, or filesystem
access need a security review.
